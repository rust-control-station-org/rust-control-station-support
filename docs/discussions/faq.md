# よくある質問（FAQ）

## Cloud と Self Hosted の違いは何ですか？

**Cloud** は、サービス提供者が運用する環境を利用する形態です。サーバーの構築や保守の負担を抑えて利用できます。

**Self Hosted** は、ご自身または所属組織が管理するサーバーに Rust Control Station を構築して運用する形態です。インフラストラクチャーの設定、更新、バックアップなどは利用者側で管理します。

利用できる機能や契約条件の詳細は、公開後の公式ドキュメントをご確認ください。

## Issue と Discussion の違いは何ですか？

**Issues** は、再現可能な不具合など、対応状況を追跡する必要がある内容に使用します。

**Discussions** は、使い方の質問、機能要望についての意見交換、利用事例や情報の共有に使用します。迷った場合は Discussion の **Q&A** でご相談ください。

## バグを報告するにはどうすればよいですか？

リポジトリの **Issues** を開き、**Bug Report** テンプレートを選択してください。発生した事象、再現手順、期待した動作、利用環境、バージョンを記載してください。

ログやスクリーンショットを添付する場合は、アクセストークン、パスワード、個人情報などが含まれていないことを必ず確認してください。

## 機能要望を送るにはどうすればよいですか？

Discussions の **Ideas** カテゴリから投稿してください。同じ提案がないか検索し、現在困っていること、提案内容、期待する効果を記載してください。既存の提案がある場合は、その Discussion にコメントやリアクションを追加してください。

## Discord でサポートを受けられますか？

Discord は準備中です。公式の案内が公開されるまでは、質問や相談には Discussions の **Q&A** をご利用ください。Discord が利用可能になった場合は、README と **Announcements** でお知らせします。

## 最新版はどこで確認できますか？

このリポジトリの **Releases** で、最新バージョンと Release Notes を確認できます。プレリリースには GitHub 上でその旨が表示されるため、安定版が必要な場合はプレリリースではない最新版を選択してください。

詳しくは[リリース運用について](release-policy.md)をご確認ください。
